# Release checklist

## Critical gates

- [ ] Product/package/trademark name cleared; `CodeSpace` and `cse` collisions resolved.
- [ ] `cargo fmt --all -- --check` passes on Rust 1.97.1.
- [ ] `cargo clippy --all-targets -- -D clippy::correctness` passes.
- [ ] `cargo test --all-targets` passes on Linux, macOS, and Windows.
- [ ] MSRV 1.85.0 check and tests pass.
- [ ] `python3 scripts/self_test.py` passes and report is attached.
- [ ] MCP initialize/tools/list/tools/call tests pass against at least two client implementations.
- [ ] Release archives and SHA-256 files are generated by CI from the signed tag.
- [ ] No credentials, `.env` files, private keys, or proprietary fixture code are present.
- [ ] License, NOTICE, third-party provenance, and SBOM reviewed.

## Important gates

- [ ] Cold and warm benchmarks include raw data and repository commit.
- [ ] Impact analysis is checked against a labeled change set.
- [ ] Secret redaction corpus contains positive and negative cases.
- [ ] Upgrade and index-corruption recovery are tested.
- [ ] Changelog states limitations and migration behavior.

## Publication decision

A source archive is not a verified binary release until all critical Cargo/CI gates have passed. Do not publish token-savings or speed multipliers without the benchmark evidence defined in `docs/BENCHMARKS.md`.
